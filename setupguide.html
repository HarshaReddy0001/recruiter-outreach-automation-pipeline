Install PythonWindows — the exact clicks, in order

ℹ️
If you already have Python installed, open Command Prompt and run python --version. If it shows Python 3.8.x or higher — skip to Step 03.
  1. Go to python.org
    Open your browser → go to python.org/downloads → click the big yellow "Download Python 3.x.x" button.
  2. Run the installer
    Open your Downloads folder → double-click the file (named something like python-3.12.0-amd64.exe).
  3. ⚠️ CHECK "Add Python to PATH" FIRST
    On the installer's first screen — at the very bottom — you will see a checkbox. Check it before clicking anything else. This is the #1 mistake beginners make. Without this, Python won't work from the terminal.
  4. Click "Install Now"
    Click "Install Now" (not "Customize"). Wait for it to finish. Click Close when done.
  5. Verify the install
    Press Win + R → type cmd → press Enter. In the black window, type:
CMD
python --version
✅
If you see Python 3.x.x — Python is installed correctly. Move to Step 03.
If you see 'python' is not recognized — you missed the PATH checkbox. Uninstall Python and reinstall, making sure to check it.

Create Virtual EnvironmentIsolate project dependencies from your system Python

ℹ️
A virtual environment is a private, isolated copy of Python just for this project. It prevents conflicts with other Python projects on your computer. Always use one.

Make sure VS Code's integrated terminal is open and you are inside the google_leads_extractor folder. Then run:

VS Code Terminal
# Step 1: Create the virtual environment
python -m venv .venv

# Step 2: Activate it
.venv\Scripts\activate
PS ...\google_leads_extractor> python -m venv .venv
PS ...\google_leads_extractor> .venv\Scripts\activate
(.venv) PS ...\google_leads_extractor> _
✅
When you see (.venv) at the start of your terminal prompt — the virtual environment is active and working.
⚠️
Every time you open a new VS Code terminal window, you must run .venv\Scripts\activate again before running any scripts. The activation does not persist between sessions.

Select the Virtual Environment in VS Code

  1. Press Ctrl+Shift+P to open the Command Palette
  2. Type "Python: Select Interpreter" and press Enter
  3. Select the one that shows .venv — it will look like ('.venv': venv) ./.venv/Scripts/python.exe

This tells VS Code to always use your project's Python, not the system one.

08

Gmail API SetupThe most detailed step — done once, works forever

💡
This section has 5 sub-steps. Read through all of them once before starting. Each one flows into the next. You only need to do this one time — after that, the token.json handles everything automatically.

8.1 — Create a Google Cloud Project

  1. Go to console.cloud.google.com
    Sign in with the Gmail account you want to use for sending emails.
  2. Click the project dropdown at the top
    It shows "Select a project" or a current project name. Click it.
  3. Click "New Project" in the popup (top right)
  4. Name it anything
    Example: leads-outreach. Leave Location as "No organization". Click Create.
  5. Make sure you're inside the new project
    Click the project dropdown again and select your new project. The name should now show at the top of the page.

8.2 — Enable the Gmail API

  1. In the top search bar, type "Gmail API" and press Enter
  2. Click "Gmail API" in the results
  3. Click the blue "Enable" button
    Wait a few seconds. You should see a green confirmation or "API enabled". ✅

8.3 — Configure the OAuth Consent Screen

⚠️
This step is required before you can create credentials. Google forces you to set up a consent screen — even for private personal apps.
  1. In left sidebar → APIs & Services → OAuth consent screen
  2. Select "External" → click Create
  3. Fill in required fields only
    App name: anything (e.g. Leads Outreach)
    User support email: select your email
    Developer contact info: type your email
  4. Click "Save and Continue" → on Scopes page, DON'T add anything → "Save and Continue"
  5. On "Test Users" → click "+ Add Users" → type your Gmail address → Add → "Save and Continue"
  6. Click "Back to Dashboard"

8.4 — Create OAuth 2.0 Credentials

  1. Left sidebar → APIs & Services → Credentials
  2. Click "+ Create Credentials" → "OAuth client ID"
  3. Application type → select "Desktop app"
  4. Name → type anything, e.g. leads-client → click Create
  5. In the popup that appears → click "Download JSON"
  6. Rename the downloaded file
    It will have a long name like client_secret_123...json. Rename it to exactly: credentials.json (all lowercase, no spaces).
  7. Move credentials.json to your project root
    Place it here: google_leads_extractor\credentials.json — same folder as README.md.

8.5 — First-Time Gmail Authorization

This connects your Gmail account to the app. You do this once. It creates token.json which the app uses automatically from then on.

In VS Code terminal (with (.venv) active), run:

VS Code Terminal
python src\gmail_auto_sender.py
  1. A browser window opens automatically
    If it doesn't, a URL will appear in the terminal — copy-paste it into your browser.
  2. "Google hasn't verified this app" warning appears
    This is completely normal. It's your own private app. Click "Advanced" → then "Go to [app name] (unsafe)".
  3. Click "Continue" on the permissions screen
  4. Browser shows success message
    You'll see "The authentication flow has completed" or similar. You can close the browser tab.
  5. Check for token.json in your project folder
    Go to VS Code Explorer panel — you should now see token.json in the root folder. This file = Gmail is authorized. ✅
✅
You now have both credentials.json and token.json in your project root. Gmail is configured. You will never need to do this again — unless you delete token.json.

Add Serper API Key to ScriptEdit google_leads_serper_fetch1.py directly in VS Code

The Serper API key is hardcoded directly inside the script file. Open it in VS Code and paste your key.

  1. Open the file in VS Code
    In VS Code's left Explorer panel → click google_leads_serper_fetch1.py to open it.
  2. Find the API key line
    Press Ctrl+F to open Find → search for SERPER_API_KEY. You'll find a line like:
google_leads_serper_fetch1.py — BEFORE
SERPER_API_KEY = 'YOUR_API_KEY_HERE'
  1. Replace the placeholder with your real key
    Delete YOUR_API_KEY_HERE and paste your Serper key from Step 07.
  2. Save the file
    Press Ctrl+S to save. VS Code shows a dot on the tab when unsaved — make sure it's gone.
google_leads_serper_fetch1.py — AFTER
SERPER_API_KEY = 'a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4e5f6a1b2'
⚠️
Never push this file to GitHub with the real key in it. Add google_leads_serper_fetch1.py to .gitignore or replace the key with a placeholder before committing.

Also: Set the File Path in extract_contacts.py

The LinkedIn extractor needs to know where to find your linkedin.txt file. Open output5\linkedin_extractor\extract_contacts.py in VS Code and find the file path variable. It will look like:

extract_contacts.py
# Find this line and set the correct path to your linkedin.txt
FILE_PATH = r"output5\linkedin_extractor\linkedin.txt"

# If your project is on the Desktop, the full path would be:
FILE_PATH = r"C:\Users\YourName\Desktop\google_leads_extractor\output5\linkedin_extractor\linkedin.txt"
💡
Windows Path Tip: To get the exact full path of any file — open File Explorer → navigate to the file → hold Shift + right-click the file → click "Copy as path". Paste that into the script. Add an r before the quotes to make it a raw string: r"C:\Users\..."

LinkedIn Manual CollectionThe human-powered step — do this before every pipeline run

⚠️
This step is done by YOU, not the code. LinkedIn aggressively blocks automated scraping. This manual approach keeps your account safe while still finding real recruiter emails from public posts.

Step 10.1 — Log into LinkedIn

Open your browser → go to linkedin.com → log in to your account.

Step 10.2 — Use Boolean Search Queries

Click in the LinkedIn search bar at the top. Paste one of these carefully crafted Boolean queries, then press Enter:

🔥 Best Results
("email me" OR "send your resume") AND ("Data Scientist" OR "Data Science")
🤖 AI / ML Focus
("hiring gen AI") ("@") ("send resume" OR "email") -senior -sr
🌎 USA Targeted
("hiring gen AI") AND ("USA") AND ("@" OR "email")
🏢 Recruiter Find
"Technical Recruiter" "Data Science" "Google" "USA"
🧑‍💻 AI Engineer
("hiring data scientist" OR "hiring ai engineer") ("@") ("send resume" OR "email")
#️⃣ Hashtag Search
#hiring #datascientist #usa
#hiring #AIengineer #usa
#genai #usa #jobs

Step 10.3 — Filter by Posts + Recent Date

  1. After search results appear — click "Posts" tab
    Look below the search bar for tabs: All, People, Jobs, Posts. Click Posts.
  2. Filter by recent date
    Find the "Date posted" filter → select "Past week" or "Past 24 hours" for fresh, active recruiter leads.

Step 10.4 — Expand Posts and Read Content

  1. Scroll through results
    Look for posts that mention "email me", contain an @ symbol, or talk about hiring with resume submission instructions.
  2. Click "...see more" on each relevant post
    This expands the full post text — email addresses are often in the part that gets cut off. Do this for 30–50 posts.

Step 10.5 — Copy All Visible Content

  1. Scroll back to the top of the page
  2. Click once in an empty area of the page (not on a button)
  3. Press Ctrl+A to select everything
  4. Press Ctrl+C to copy

Step 10.6 — Save as linkedin.txt

  1. Open Notepad
    Press Win+R → type notepad → press Enter.
  2. Paste the copied content
    Press Ctrl+V.
  3. Save the file to the correct location
    Click File → Save As. Navigate to: google_leads_extractor → output5 → linkedin_extractor. Name it linkedin.txt. Make sure "Save as type" is "Text Documents (*.txt)" — not "All Files".
⚠️
Windows Double Extension Trap: Notepad sometimes saves linkedin.txt.txt instead of linkedin.txt. To check: open File Explorer → View → check "File name extensions". If you see linkedin.txt.txt, rename it to remove the extra .txt.

TroubleshootingEvery common error — cause and exact fix

❌ 'python' is not recognized as a command

🔧
Cause: Python not added to PATH during install.
Fix: Uninstall Python from Control Panel → reinstall → on the first screen, check "Add Python to PATH" before anything else.

❌ ModuleNotFoundError: No module named 'requests'

🔧
Cause: Running Python without the virtual environment active, OR packages weren't installed.
Fix: Run .venv\Scripts\activate first → then pip install -r requirements.txt.

❌ FileNotFoundError: linkedin.txt not found

🔧
Cause: File is missing, saved with wrong name (linkedin.txt.txt), or path in script is wrong.
Fix: Check File Explorer with extensions visible. Rename if needed. Update the FILE_PATH variable in extract_contacts.py to the exact full path.

❌ credentials.json not found

🔧
Cause: File renamed incorrectly or placed in wrong folder.
Fix: File must be named exactly credentials.json and live in the same folder as README.md.

❌ "Google hasn't verified this app" (Browser)

ℹ️
This is expected and safe. It's your own private app. Click Advanced → "Go to [app name] (unsafe)" → Continue. This is not a real security risk.

❌ Token has been expired or revoked

🔧
Cause: token.json expired.
Fix: Delete token.json from the project folder → run python src\gmail_auto_sender.py again → re-authorize in browser.

❌ leads_emails.txt is empty

🔧
Cause: The posts you copied from LinkedIn didn't contain visible email addresses.
Fix: Use better Boolean queries (especially the ones with "@"). Expand more posts before copying. Make sure you click "See more" on individual posts before Ctrl+A.

❌ Gmail sends 0 emails / skips everyone

🔧
Cause: Everyone is already in sent_log.txt from a previous run (prevents duplicates — this is by design).
Fix for testing: Open output5\sent_log.txt → delete all content → save the empty file → run again.

Security ChecklistRun through this before your first real send

  • token.json is in .gitignore — will NOT be pushed to GitHub
  • credentials.json is in .gitignore — will NOT be pushed to GitHub
  • Serper API key is not in a public file — consider removing before pushing
  • Email template reviewed — open gmail_auto_sender.py and confirm the email body, subject, and your name are correct
  • Tested on your own email first — add yourself to all_emails.txt, run the sender, confirm you receive it properly
  • Daily send limit is conservative — stay under 100 emails/day to avoid Gmail spam flags
  • Only targeting public posts — you are emailing people who shared their email address publicly in a LinkedIn post
